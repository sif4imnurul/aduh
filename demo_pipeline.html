GET /grid-agenda

AgendaController@grid
45.27 mWh
Diagnosis: Terdeteksi pengambilan data tanpa paginasi. Pemakaian memori sangat tinggi dan eksekusi query lambat akibat ->get(). Proses loop berlanjut menghasilkan kompleksitas berat O(N*M).
Kode Asli
public function grid(Request $request)
{
    $month = $request->input('month', Carbon::now()->month);
    $year = $request->input('year', Carbon::now()->year);
    
    // Inefficient read
    $agendas = Agenda::whereMonth('tanggal_mulai', $month)
                    ->whereYear('tanggal_mulai', $year)
                    ->orWhere(function($query) use ($month, $year) {
                        $query->whereMonth('tanggal_deadline', $month)
                              ->whereYear('tanggal_deadline', $year);
                    })->get();
    
    $calendar = $this->generateCalendar($month, $year, $agendas);
    return view('pages.reference.agenda.grid', compact('calendar', 'agendas'));
}
Saran Optimasi
public function grid(Request $request)
{
    $month = $request->input('month', Carbon::now()->month);
    $year = $request->input('year', Carbon::now()->year);
    
    $startOfMonth = Carbon::create($year, $month, 1)->startOfDay();
    $endOfMonth = Carbon::create($year, $month, 1)->endOfMonth()->endOfDay();

    // Range bounded query
    $agendas = Agenda::where(function($query) use ($startOfMonth, $endOfMonth) {
                        $query->where('tanggal_mulai', '<=', $endOfMonth)
                              ->where('tanggal_deadline', '>=', $startOfMonth);
                    })->get();

    $agendasGroupedByDay = collect();
    foreach ($agendas as $agenda) {
        // O(1) structure logic
    }
    
    $calendar = $this->generateCalendar($month, $year, $agendasGroupedByDay);
    return view('pages.reference.agenda.grid', compact('calendar', 'agendas'));
}
Perubahan Logika: Parameter query dibatasi dan menggunakan penyusunan data tingkat lanjut untuk akses kecepatan O(1).
Dampak (Delta): Penurunan drastis pemakaian siklus CPU dengan mencegah loop berulang pada perenderan tampilan.
Penjelasan Teknis:
  1. Mengganti pencocokan tanggal luas dengan perbandingan timestamp yang ketat.
  2. Mengaplikasikan memori kelompok (*memory grouping*) untuk mencegah loop berlebih saat kalender dimuat.

GET /logo

LogoController@index
42.22 mWh
Diagnosis: Tarikan database langsung secara keseluruhan tanpa adanya batasan (*offset mapping*), mengakibatkan RAM membengkak.
Kode Asli
public function index()
{
    $logos = ProdukModel::where('kategori', 'logo')->get();
    return view('pages.picture.logo.index', compact('logos'));
}
Saran Optimasi
public function index()
{
    // Limit application
    $logos = ProdukModel::where('kategori', 'logo')->paginate(15);
    return view('pages.picture.logo.index', compact('logos'));
}
Perubahan Logika: Implementasi batas paginasi otomatis.
Dampak (Delta): Konsumsi memori menjadi jauh lebih ringan dan stabil pada bobot *payload* tetap.
